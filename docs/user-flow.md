# 用户行为流程

本文描述最终用户在柜机前的完整交互流程。

## 正常流程
1. 用户进入首页，选择目标门位（A01~A08）。
2. 页面提示“请将校园卡贴近读卡区”。
3. 系统读取卡 UID，随后进入“正在验证身份”。
4. 上级返回允许后，设备执行开门脉冲。
5. 页面显示“已开门，请取物并关门”，用户完成后点击确认。
6. 会话结束，页面返回首页。

## 异常分支
- 鉴权拒绝（业务码非 0）：
  - 不开门。
  - 页面展示拒绝原因。
  - 用户可“重试”或“返回”。
- 网络异常（超时/非 2xx/解析失败）：
  - 不开门。
  - 页面提示“网络异常，暂不可开门”。
  - 用户可“重试”或“返回”。
- 开门后未确认完成：
  - 到达超时时间后自动结束会话。
  - 页面自动返回首页。

## 交互约束
- 去抖策略：同卡同门在短时间内重复刷卡会被忽略，避免重复鉴权。
- 安全策略：断网不放行，网络失败与业务拒绝统一走“不打开门”。
